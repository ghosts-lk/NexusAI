## How We Ensure Compliance

1. **Dependency Audit**: Run `npm audit` to check for vulnerabilities
2. **License Checking**: Verify all licenses are compatible with MIT
3. **Attribution**: List all dependencies and their licenses
4. **Updates**: Keep dependencies current for security
5. **No Proprietary Code**: Never include proprietary libraries

### Legal Review Checklist

- ✅ All dependencies use compatible licenses (MIT, Apache 2.0, ISC)
- ✅ No GPL or AGPL (except Supabase which is AGPL)
- ✅ Proper attribution in ATTRIBUTIONS.md
- ✅ No copyrighted assets included
- ✅ Original code licensed under MIT
- ✅ No proprietary SDKs embedded

## Contributing with Proper Attribution

When contributing to NexusAI:

1. **Use open-source libraries only**
   - Check license before adding dependency
   - Ensure compatibility with MIT

2. **Attribute external code**
   - Add to ATTRIBUTIONS.md
   - Include license information
   - Link to original source

3. **Create original code**
   - Don't copy code from proprietary sources
   - Rewrite or use properly licensed alternatives

Example of proper attribution:

```typescript
/**
 * Based on [Original Project Name]
 * Licensed under [License Type]
 * https://github.com/username/project
 * 
 * Modified by Ghost Protocol (Pvt) Ltd
 */
```
